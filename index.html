About the Project

What Problem Are We Solving?

⚠️

The Problem

Bus luggage compartment doors are often left unsecured, causing baggage to fall and potentially creating road hazards. There's no automated way to monitor these doors in real time.

🔬

Our Solution

An ESP32-based embedded system using vibration sensors, magnetic reed switches, and a camera module to monitor door state 24/7 and alert the driver immediately if any anomaly is detected.

🎯

Impact

Prevents luggage loss and road accidents, improves fleet safety, provides actionable data for maintenance teams, and is scalable across any bus fleet at minimal cost.

Vibration Sensor
MPU6050 — Accelerometer + Gyroscope

The MPU6050 is a 6-axis motion tracking chip that detects acceleration and angular velocity. It captures vibration amplitude and frequency to identify abnormal door behaviour, loose latch, or vibrations from an impact.

InterfaceI²C
Voltage3.3V / 5V

Magnetic Reed Switch

A magnetic reed switch uses a permanent magnet attached to the door and a switch mounted on the frame. When the magnet is in proximity, the switch closes, indicating the door is closed. Separation signals an open state.

Safety System

Alert System

The alert system is the safety backbone of the monitoring system. When sensor data crosses defined thresholds, alerts are triggered through multiple output channels to ensure the driver and fleet manager are immediately informed.

How It Works

Project Working

The system continuously monitors the luggage compartment door using multiple sensors operating in parallel. The ESP32 microcontroller reads data from the MPU6050 vibration sensor and the magnetic reed switch every 500ms.

If abnormal vibration (above threshold) or door opening is detected while the bus is in motion, the system immediately triggers the buzzer and LED indicators to alert the driver. Simultaneously, the event is logged to the web dashboard with a timestamp.

The ESP32-CAM captures a photograph of the door condition during any fault event, providing visual evidence for maintenance records. All data is transmitted over WiFi to the monitoring dashboard.

Operation Flow
1
Power On & Init
ESP32 boots, connects WiFi, initializes all sensors
2
Continuous Sensing
Read reed switch & MPU6050 every 500ms
3
Threshold Check
Compare readings against normal/warning/critical thresholds
4
Fault Detected
Trigger buzzer, LED, capture camera image, send push alert
5
Log & Report
Event stored in dashboard with timestamp & sensor readings

Real-Time Detection

Sensor readings are processed every 500ms ensuring sub-second fault detection and alerting.

📡

WiFi Connectivity

Built-in WiFi on ESP32 enables wireless data transmission to the dashboard and mobile devices.

💾

Event Logging

Every fault event is logged with timestamp, sensor values, and camera snapshot for maintenance records.
